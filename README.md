
# Algorithm for the Traveling Salesman Problem (TSP)

The goal of the Traveling Salesman Problem is to find a minimum path that visits all cities once, minimizing the total distance.

## Fast algorithm
For the Fast algorithm I used a greedy approach that creates an edge between the actual city and the nearest city not visited yet.

## Slow algorithm

## 1. Population Initialization
A population is initialized with several random paths, each represented as an individual. Also the fast algorithm solution is added to the population to provide an already optimized baseline.

## 2. Fitness Function
- **Cost Calculation**: Each path has a total cost calculated as the sum of distances between cities, considered as pairs of nodes in a graph. The path with the lowest cost is the best path.
- **Ordering Individuals**: Individuals in the population are sorted by cost, and the path with the lowest cost is considered the current best candidate.

## 3. Evolutionary Cycle
The evolutionary cycle repeats for a set number of iterations. In each iteration:

### Parent Selection
- **Cost-Based Selection**: A subset of the population is selected, favoring lower-cost paths.
- **Parent Diversification**: To avoid repeated solutions, the two selected parents for crossover are chosen randomly among the best candidates, ensuring they differ.

### Crossover and Mutation
- **Inver-Over Crossover Probability**: With a certain probability, the algorithm applies the "inver-over" crossover to the selected parents. 
- **Inversion Mutation**: If crossover is not applied, the path is mutated by reversing the order of cities between two random positions. This permits to preserve the "element near to another element" trait.

### Adding Local Search
After crossover and mutation, a **local search** step is applied to further optimize some selected individuals, seeking improvements in total cost.

## 4. Population Update
- **Population Extension**: The population is extended with the newly generated paths.
- **Population Reduction**: Individuals are sorted by cost, and only the best are retained for the next generation.

## 5. Progress Control
- **Progress Control**: If no improvements in cost are observed for a certain number of iterations, the algorithm stops and returns the best path found up to that point.
